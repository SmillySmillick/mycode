## 二、LeetCode100 已覆盖的主要算法标签

`leetcode100` 里最集中的标签是：

- 动态规划：爬楼梯、杨辉三角、最长公共子序列、单词拆分、打家劫舍、不同路径、最小路径和、编辑距离、最大正方形等。
- 树形 DP：二叉树直径、二叉树最大路径和、打家劫舍 III。
- 背包：分割等和子集、目标和、零钱兑换。
- 贪心：跳跃游戏、跳跃游戏 II、任务调度器、划分字母区间。
- 单调结构：每日温度、柱状图最大矩形、滑动窗口最大值。
- 双指针/滑动窗口：三数之和、接雨水、最小覆盖子串、无重复字符最长子串。
- 树与图：BFS、DFS、拓扑排序、二叉树递归、BST。
- 堆与哈希：Top K、中位数、LRU、前缀和哈希。

缺口主要是：线段树、树状数组、并查集、树链剖分、莫队、最短路/MST/匹配、数论提高。这些更偏 ACM，不是 Hot 100 的重点。

## 三、重点一：动态规划 DP

DP 的本质是“用状态保存子问题答案，再按依赖顺序转移”。遇到题目时不要先背公式，先回答四个问题：

1. 状态是什么：`dp[i]`、`dp[i][j]`、`dp[u]` 分别代表什么。
2. 转移从哪里来：当前状态由哪些更小或更早的状态推出来。
3. 初始化是什么：空串、边界行列、叶子节点、容量 0 等。
4. 遍历顺序是什么：必须保证转移来源已经算好。

### 1. 线性 DP

适合一维顺序结构。

- `70. 爬楼梯`：`dp[i] = dp[i - 1] + dp[i - 2]`。
- `198. 打家劫舍`：第 `i` 间偷或不偷，`dp[i] = max(dp[i - 1], dp[i - 2] + nums[i])`。
- `53. 最大子数组和`：`dp[i]` 表示以 `i` 结尾的最大和，必须包含 `nums[i]`。
- `152. 乘积最大子数组`：负数会翻转最大/最小，所以同时维护 `maxDp` 和 `minDp`。

线性 DP 的关键是状态含义要“闭合”。例如最大子数组不能只说“前 i 个数最大和”，否则不知道是否能接上 `nums[i]`。

### 2. 二维 DP

适合两个序列、网格、区间、字符串匹配。

- `1143. 最长公共子序列`：`dp[i][j]` 表示 `text1[0..i)` 和 `text2[0..j)` 的 LCS 长度。
- `72. 编辑距离`：`dp[i][j]` 表示把 `word1[0..i)` 变成 `word2[0..j)` 的最少操作数。
- `62. 不同路径`、`64. 最小路径和`：网格 DP，转移来自上方和左方。
- `221. 最大正方形`：如果当前位置是 `1`，边长由左、上、左上三个状态的最小值决定。

二维 DP 常见初始化是第 0 行、第 0 列。很多错误都来自边界没有定义清楚。

### 3. 背包 DP

背包是“容量限制下做选择”。Hot 100 中出现了 01 背包和完全背包。

- `416. 分割等和子集`：是否能选出和为 `sum / 2` 的子集，是 01 背包可行性问题。
- `494. 目标和`：可转化为选若干数使其和为 `(sum + target) / 2`。
- `322. 零钱兑换`：每种硬币可无限选，是完全背包最少数量问题。

遍历顺序：

- 01 背包：容量倒序，避免同一物品被重复使用。
- 完全背包：容量正序，允许当前物品重复使用。

### 4. 树形 DP

树形 DP 的状态通常定义在节点上，递归返回“以当前节点为根的子树贡献”。

- `543. 二叉树直径`：递归返回单边最大深度，用左右深度更新答案。
- `124. 二叉树最大路径和`：递归返回“向父节点延伸的最大贡献”，答案可用左右两边同时经过当前点更新。
- `337. 打家劫舍 III`：每个节点返回两个值：偷当前节点、不偷当前节点。

树形 DP 的关键是区分“返回给父亲的贡献”和“全局答案”。有些路径可以在当前节点拐弯，但拐弯后的路径不能继续向父节点延伸。

### 5. 状压 DP 与提高内容

`第9章-动态规划提高.pdf` 涉及树形 DP、树上背包、换根 DP、状压 DP、高维前缀和。它们可以作为 Hot 100 之后的进阶：

- 状压 DP：用二进制集合表示已选择状态，适合 `n <= 20` 左右的问题。
- 换根 DP：先自底向上算子树答案，再自顶向下把父侧贡献传给孩子。
- 树上背包：每个节点有容量/选择限制，合并子树状态。

## 四、重点二：贪心

贪心不是“看起来每步最优”，而是要能证明“当前局部选择不会破坏全局最优”。常用证明方式有：

- 交换论证：如果最优解没有选当前贪心选择，可以把它换成当前选择，答案不变差。
- 覆盖论证：每一步尽量扩大覆盖范围，步数或代价不会更差。
- 排序后选择：先按结束时间、左端点、右端点、频率等关键字排序，再做局部决策。

### 1. 覆盖型贪心

- `55. 跳跃游戏`：维护当前能到达的最远位置 `farthest`，只要遍历过程中 `i <= farthest`，就能继续扩展。
- `45. 跳跃游戏 II`：把每次跳跃看成 BFS 的一层，在当前层内维护下一层最远边界，走到层尾就必须跳一次。

这类题的核心是“当前步数能覆盖的范围”和“下一步能覆盖的最远范围”。

### 2. 区间/划分型贪心

- `763. 划分字母区间`：先记录每个字符最后出现位置，扫描时维护当前段最远右边界，走到边界就切一段。
- `56. 合并区间` 虽然标签是排序，但思想也接近区间贪心：按左端点排序，能合并就扩右端点。

区间题常见套路是先排序，再维护当前区间的边界。

### 3. 计数构造型贪心

- `621. 任务调度器`：优先考虑出现次数最多的任务，用冷却间隔形成框架，再填充其他任务。
- `300. 最长递增子序列` 文件标为二分贪心：维护 `tails[len]` 为长度为 `len + 1` 的递增子序列的最小结尾。结尾越小，后续扩展空间越大。

贪心题的危险点是样例容易误导。写题解或代码前至少要说明：为什么当前选择不会让后续更差。

## 五、重点三：线段树

线段树是 ACM 提高数据结构的核心，`第6章-数据结构提高.pdf` 单独讲了基础线段树、带标记线段树、扫描线。它解决的是：

> 数组不断变化，同时要频繁查询区间信息。

典型操作：

- 单点修改 + 区间查询：例如修改一个位置的值，查询区间和/最大值/最小值。
- 区间修改 + 区间查询：例如一段区间都加 `x`，查询区间和。
- 扫描线：把二维矩形覆盖面积转化为一维区间覆盖长度的动态维护。

### 1. 线段树维护什么

每个节点维护一个区间 `[l, r]` 的信息。常见信息包括：

- `sum`：区间和。
- `mx/mn`：区间最大值/最小值。
- `cnt`：满足条件的数量。
- `len`：扫描线中被覆盖的长度。

线段树能做的前提是：父区间答案可以由左右子区间答案合并得到，也就是有明确的 `pushup`。

### 2. 基础线段树结构

常见函数：

```cpp
void build(int p, int l, int r);
void update(int p, int l, int r, int pos, int val);
int query(int p, int l, int r, int ql, int qr);
void pushup(int p);
```

`p` 是当前节点编号，左儿子通常是 `p << 1`，右儿子是 `p << 1 | 1`。数组大小一般开 `4 * n`。

复杂度：

- 建树：`O(n)`
- 单点修改：`O(log n)`
- 区间查询：`O(log n)`

### 3. 懒标记 lazy tag

当需要区间修改时，如果每次都递归到叶子，复杂度会退化。懒标记的思想是：

- 当前节点区间完全被修改区间覆盖时，直接更新当前节点答案，并记录一个 tag。
- 以后访问它的子节点前，再把 tag 下传给左右儿子。

常见函数：

```cpp
void apply(int p, int l, int r, int add);
void pushdown(int p, int l, int r);
void rangeAdd(int p, int l, int r, int ql, int qr, int add);
long long rangeQuery(int p, int l, int r, int ql, int qr);
```

区间加、区间求和时：

- `sum[p] += 1LL * (r - l + 1) * add`
- `lazy[p] += add`

### 4. 线段树与树状数组的选择

优先用树状数组的情况：

- 只需要前缀和、区间和。
- 修改和查询都比较简单。
- 需要代码短、常数小。

必须考虑线段树的情况：

- 维护最大值、最小值、区间覆盖长度等复杂信息。
- 需要区间修改配合区间查询。
- 需要多个信息一起维护。
- 需要扫描线、动态开点、可持久化等扩展。

### 5. 与 Hot 100 的衔接

Hot 100 没有典型线段树题，但以下知识可以迁移：

- 前缀和题训练“区间信息”的意识。
- 单调栈/单调队列训练“维护候选结构”的意识。
- 二分和排序训练“把问题转成可维护区间”的意识。

从 Hot 100 过渡到线段树，可以按这个顺序刷：

1. 区间和查询 - 数组可修改。
2. 区间最小值/最大值查询。
3. 区间加、区间求和。
4. 扫描线求矩形面积并。
5. 动态开点线段树。

## 六、建议学习路线

1. 先用 `leetcode100` 巩固基础结构：哈希、栈、队列、链表、二叉树、堆、双指针、滑动窗口。
2. 集中突破 DP：线性 DP -> 二维 DP -> 背包 -> 树形 DP -> 状压/换根。
3. 集中突破贪心：覆盖、区间、排序构造，每题都写出证明思路。
4. 回到 ACM 课件第 6 章：线段树 -> 树状数组 -> 并查集 -> 分块/莫队。
5. 再补图论与数学提高：最短路、MST、Tarjan、匹配、数论函数、CRT、Lucas。
